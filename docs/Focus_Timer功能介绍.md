# Focus Timer 功能介绍

更新时间：2026-06-22

## 一、产品定位

Focus Timer 是一个 Android 优先、本地优先的效率工具，核心目标是把「待办计划」「专注计时」「统计复盘」「本地数据管理」放在一个轻量应用里完成。

它不是社区型、云同步型、会员型工具；当前方向更接近一个可迁移、可离线、可长期保存数据的个人效率系统。

当前主导航为四个板块：

- 待办
- 计时
- 统计
- 我的

应用整体设计原则：

- 数据尽量保存在本地，减少账号、云服务、第三方依赖。
- 待办负责“要做什么”，计时负责“做了多久”，统计负责“回看与复盘”。
- 视频凭证、导出 JSON、打卡日志等都尽量放进可迁移的数据闭环。
- Android APK 是当前第一优先级，Web/桌面端暂不作为重点。

## 二、待办模块

待办模块用于管理任务、分组、计划模式、完成记录和任务属性。

### 1. 分组管理

待办支持多个分组，每个分组可以理解为一个项目、清单或任务池。

已支持能力：

- 创建分组
- 重命名分组
- 修改分组颜色
- 修改分组图标
- 删除分组到回收站
- 恢复回收站分组
- 永久删除分组
- 删除空分组
- 删除所有分组
- 分组列表/宫格视图切换
- 分组日倒计时展示

分组可以用于区分学习、工作、生活、考试、习惯等不同任务类型。

### 2. 任务管理

每个分组内可以创建具体待办任务。

已支持能力：

- 新建待办
- 重命名待办
- 完成/取消完成
- 删除待办
- 删除后撤回
- 移动到其他分组
- 复制待办
- 批量选择任务
- 批量完成
- 批量移动
- 批量删除
- 拖拽排序
- 左滑删除

任务详情字段：

- 标题
- 备注
- 优先级
- 截止日期
- 预计时长
- 重点标记
- 独立任务颜色
- 置顶
- 提醒时间
- 重复规则

### 3. 重点与置顶

当前任务支持两种“重要性”表达：

- 重点：表示这个任务值得关注。
- 置顶：表示这个任务要排在列表前面。

两者用途不同：

- 重点更像标签，方便后续筛选和统计。
- 置顶更像排序规则，直接影响任务在列表中的位置。

当前排序规则已接入：

1. 置顶任务优先
2. 置顶时间较新的优先
3. 高优先级优先
4. 手动排序

### 4. 提醒与重复任务

当前已经接入任务提醒和重复任务的数据结构与基础逻辑。

提醒能力：

- 在任务详情中设置提醒日期和时间
- 到点由 Android 原生提醒通知触发
- 完成任务后取消当前任务提醒

重复规则：

- 不重复
- 每天
- 每周
- 每月

重复任务逻辑：

- 当一个重复任务被标记完成时，系统会自动创建下一条未完成任务。
- 下一条会继承标题、备注、优先级、预计时长、重点、颜色、重复规则。
- 如果原任务有提醒，会迁移到下一周期。

注意：

- 不同 Android 系统对“精确提醒/闹钟与提醒”权限处理不同，后续需要增加一个权限状态提示页。

### 5. 已办视图

待办首页支持在“待办”和“已办”之间切换。

已办视图用于查看已经完成的任务。

已支持能力：

- 查看已完成任务
- 按分组查看完成记录
- 取消完成
- 重命名
- 复制
- 移动分组
- 设为重点/取消重点
- 置顶/取消置顶
- 删除

### 6. 搜索与筛选

当前搜索已经不再只是空壳，支持展示搜索结果。

已支持：

- 搜索待办标题
- 搜索结果区分未完成和已完成
- 展示任务元信息，例如重点、优先级、日期、预计时长

后续适合补充：

- 重点筛选
- 置顶筛选
- 有提醒筛选
- 重复任务筛选
- 今天任务筛选
- 逾期任务筛选
- 已计划任务筛选

### 7. 计划模式

计划模式用于把待办任务安排到一天中的具体时间段。

已支持：

- 从任务池拖拽任务到时间轴
- 24 小时时间轴
- 自定义开始时间、结束时间、分段
- 同一时段可放多个不同任务
- 同一任务同一时段去重
- 拖动任务调整时段
- 修改计划时长
- 删除计划项
- 一键清空计划
- 从前一天/上周同一天复制计划

设计目标：

- 左侧是任务池，右侧是一天的时间安排。
- 任务不是被“消耗”，而是可以被安排到时间段。
- 适合做学习计划、工作排程、每日安排。

## 三、计时模块

计时模块用于记录实际投入时间，是统计复盘的数据来源之一。

### 1. 计时类型

当前支持多种计时：

- 自由计时
- 番茄计时
- 倒计时
- 快速休息
- 视频打卡

计时类型说明：

- 自由计时：无目标时长，开始后持续记录。
- 番茄计时：默认固定时长，适合专注周期。
- 倒计时：用户指定目标时长。
- 快速休息：一键开始 5/10/20 分钟休息。
- 视频打卡：带视频凭证的计时记录。

### 2. 运行状态

计时支持：

- 开始
- 暂停
- 继续
- 停止
- 保存记录
- 放弃记录
- 后台前台服务留存
- 计时完成通知
- 提前提醒
- 屏幕常亮

后台能力：

- 运行中可以启动 Android 前台服务。
- 应用退到后台后仍能保留当前计时会话。
- 重新进入应用后可以恢复运行中的计时。

### 3. 计时记录

计时结束后会生成记录。

记录字段：

- 名称
- 类型
- 日期
- 开始时间
- 结束时间
- 时长
- 备注
- 视频凭证 URI
- 视频文件名
- 视频相对路径
- 视频 MIME 类型

已支持记录操作：

- 继续计时
- 重命名
- 修改时长
- 修改开始/结束时间
- 修改类型
- 修改日期
- 编辑备注
- 添加/清除视频凭证
- 播放视频凭证
- 复制视频位置
- 删除记录

### 4. 视频打卡与视频凭证

视频凭证用于证明某次学习或专注确实发生过。

当前设计：

- 可以现场拍摄视频
- 可以选择已有视频
- 视频会复制到 `Download/Focus Timer/Videos`
- 计时记录保存视频索引
- JSON 导出会包含视频凭证索引
- 应用内可以直接播放视频

为什么这样做：

- 只保存路径会割裂，用户还要去文件管理器找。
- 只存在应用私有目录不方便迁移。
- 放在 Download 下的专用文件夹，用户更容易备份和转移。
- JSON 里保存索引，方便后续恢复和定位。

注意：

- 当前应用内播放使用 Android 原生 `VideoView`。
- 如果后续遇到兼容性或格式问题，可以再考虑接入 Flutter `video_player`。

### 5. 计时设置

当前计时设置包含：

- 新一天开始时间
- 完成通知
- 通知声音
- 振动
- 提前提醒
- 屏幕常亮
- 时钟样式

时钟样式当前有 5 种：

- 环形
- 数字
- 翻页感
- 进度条
- 极简

用途区别：

- 环形：适合番茄和倒计时，看进度直观。
- 数字：安静、居中、适合长时间看。
- 翻页感：更接近桌面时钟。
- 进度条：横向看进度，适合屏幕常亮。
- 极简：低干扰，只突出时间。

## 四、统计模块

统计模块用于复盘专注时间、任务完成、打卡和趋势。

### 1. 范围选择

统计支持不同时间范围：

- 日
- 周
- 月
- 年
- 自定义范围

底部有悬浮范围控制条，可以切换范围、上一段、下一段。

### 2. 热力图

热力图用于显示长期专注分布。

已支持：

- 按日期显示专注强度
- 点击日期查看当天明细
- 明细包含专注总时长、记录数、打卡数、当天计时和打卡列表

### 3. 饼图

饼图用于查看不同计时类型的占比。

已支持：

- 显示不同类型计时占比
- 图例可点击
- 点击后查看该类型记录列表和总时长
- 支持同名合并方向的后续扩展

后续可补：

- 饼图扇区直接点击
- 饼图样式切换
- 排序和分组管理

### 4. 时间轴

时间轴用于按时间顺序查看一天或一段时间里的记录。

已支持：

- 计时记录时间轴
- 打卡记录时间轴
- 点击计时记录进入编辑
- 点击打卡记录进入编辑
- 补录计时记录

### 5. 打卡统计

起床、睡觉不再作为计时，而是作为 daily logs 打卡记录。

已支持：

- 起床打卡
- 睡觉打卡
- 修改打卡类型
- 修改日期和时间
- 修改备注
- 删除打卡
- 统计页展示打卡卡片
- 时间轴合并展示打卡

后续可扩展：

- 自定义习惯打卡
- 连续天数
- 缺卡提示
- 习惯趋势

### 6. 图表管理

统计页支持管理图表显示。

当前图表包括：

- 热力图
- 饼图
- 打卡
- 时间轴

支持：

- 显示/隐藏图表
- 调整图表顺序

## 五、我的模块

我的模块主要承载应用设置、本地数据、主题、视频凭证等入口。

### 1. 主题与显示

当前支持：

- 主题色设置
- 深色模式跟随系统
- 深色模式开关相关问题已记录，后续继续打磨

后续方向：

- 低配置模式
- 卡片透明度
- 默认待办视图
- 液态玻璃/高斯模糊视觉风格

### 2. 视频凭证管理

我的页面有视频凭证入口。

已支持：

- 查看带视频凭证的计时记录
- 查看文件目录、文件名、MIME 类型、URI
- 复制视频位置
- 应用内播放视频

### 3. 数据导出

导出数据页支持多个导出范围：

- 全部数据
- 导出待办
- 导出计时
- 导出打卡

导出位置：

- JSON 文件保存到 `Download/Focus Timer/Exports`
- 视频文件保存到 `Download/Focus Timer/Videos`

JSON 内容包含：

- 应用名
- 应用版本
- 数据库版本
- 导出范围
- 导出时间
- 存储路径说明
- 分组
- 待办
- 计划
- 计时记录
- 活跃计时会话
- 统计设置
- 日倒计时
- 打卡记录
- 视频凭证索引

### 4. 数据恢复

当前恢复功能已经从“预检阶段”推进到“预检 + 追加恢复”。

已支持：

- 系统文件选择器选择 JSON
- 支持多选 JSON
- 解析每个文件
- 显示可导入数据数量
- 预检后手动确认追加恢复
- 写入本机 SQLite 数据库
- 任务、计划、计时关联 ID 自动重映射
- 基于关键字段尽量去重，重复导入同一份 JSON 不会重复插入主要数据
- 恢复日倒计时、计时设置、外观设置等本地文件设置
- 恢复计划设置和统计图表设置
- 恢复后重新调度未来待办提醒

可恢复的数据：

- 分组 `taskLists`
- 待办 `tasks`
- 计划 `taskPlans`
- 计时记录 `focusTimes`
- 打卡记录 `dailyLogs`
- 计划设置 `taskPlanSettings`
- 统计图表设置 `statsSettings`
- 日倒计时 `dayCountdown`
- 计时设置 `timerSettings`
- 外观设置 `appearanceSettings`

恢复策略：

- 默认只追加，不覆盖现有数据。
- 导入分组、待办、计划后会建立旧 ID 到新 ID 的映射。
- 计时记录里的 `taskId` / `listId` / `planId` 会尽量映射到新导入的数据。
- 如果待办缺少可用分组，会自动放到“导入恢复”分组。
- 视频凭证字段会随计时记录恢复，但视频文件本身不会被复制。

还未完成：

- 导入前自动备份
- 覆盖恢复策略
- 更细的冲突提示
- 视频文件缺失校验
- `activeTimerSessions` 活跃计时会话恢复
- 恢复结果详情页

推荐后续策略：

- 覆盖恢复必须二次确认。
- 视频缺失时保留记录，但标记“视频文件未找到”。
- 恢复前自动导出一份本机备份。
- 恢复完成后提供可展开的明细报告。

## 六、Android 权限说明

当前应用涉及的权限主要服务于计时、提醒、视频和本地文件。

已使用或已声明的权限：

- 通知权限：用于计时完成、提前提醒、待办提醒。
- 振动权限：用于通知震动。
- 前台服务权限：用于后台计时留存。
- 前台服务 specialUse：用于当前计时服务。
- Wake Lock：用于计时/常亮相关能力。
- Camera：用于视频打卡拍摄。
- Schedule Exact Alarm：用于待办精确提醒。
- Android 12 及以下读取外部存储：用于兼容老系统文件读取。

没有加入的权限：

- 定位
- 联系人
- 通话
- 短信
- 日历
- 账号

原因：

- 当前产品不需要这些能力。
- 无关权限会增加用户不信任，也可能影响系统审核和安装体验。

## 八、后续优先级建议

### 第一优先级：先打包测试

必须先构建新 APK，再测本轮功能。

建议测试顺序：

1. 安装新 APK
2. 打开待办，测试置顶/提醒/重复
3. 设置一个 1-2 分钟后的提醒，确认通知是否出现
4. 创建一个每天重复任务，完成后看下一条是否生成
5. 打开计时，切换 5 种时钟样式
6. 视频打卡选择“拍摄”，完成后播放
7. 视频打卡选择“已有视频”，完成后播放
8. 我的 -> 数据导出 -> 恢复数据 -> 多选 JSON 预检
9. 确认追加恢复后检查分组、待办、计划、计时、打卡和设置是否写入

### 第二优先级：完善恢复安全性

## 九、一句话总结

Focus Timer 当前已经从单纯计时器，逐步变成一个本地优先的个人效率系统：用待办承接计划，用计时记录投入，用统计完成复盘，用导出和视频凭证保证数据可迁移、可追溯。
