# Codex 观察记录

## 2026-06-18 reference app 登录后交互巡检

本文件只记录参考应用 参考应用的交互观察和可借鉴点，不记录本项目构建、测试、修复流水。参考对象为 `releases/reference/1.18.13.apk`，包名 `reference.app`。观察方式为 Android 12 模拟器上的截图、UIAutomator 控件树和手势点击巡检；只学习交互组织，不复制代码、图片或素材。

### 当前观察状态

- 模拟器：`emulator-5556`，Android 12，竖屏 `900x1600`，density `320`。
- reference app 已登录，账号页显示 `DemoUser` / `User ID: 1423530`。
- 为观察创建过临时分组 `CodexTemp` 和任务 `codexTask`，之后已通过 reference app 自身流程移入回收站，未做永久删除。
- 点休息快捷项时产生了一条约 46 秒的休息记录，时间轴中可见。

### 清单页

- 空状态首屏很克制：左上大标题 `待办`，右上四个圆形入口分别是创建分组、回收站、计划视图、更多；中间是空状态插画卡和 `创建分组` CTA。
- 底部导航固定为 `清单 / 统计 / + / 发现 / 我`，中间 `+` 是全局动作入口，不是添加待办入口。
- 创建分组使用底部抽屉：背景压暗，抽屉标题 `创建新分组`，右侧 `创建` 按钮，下方输入框示例 `如：数学`。这个比中间弹窗更贴近移动端节奏。
- 分组卡片是瀑布流/紧凑卡：标题、任务摘要、右下三点菜单；卡片可展开/收起，任务完成后在卡内置灰显示。
- 计划视图是双栏模式：左侧说明和待办列表，右侧按时间轴排列日程；有 `退出计划模式`。空待办时仍展示完整时间刻度。
- 回收站独立页面：回收分组显示名称、回收日期、任务摘要，底部有 `彻底删除` 和 `恢复`。永久删除入口明确，适合保留二次确认。
- 清单右上更多菜单为右侧纵向浮层，包含：`全部完成`、`回收空的分组`、`回收所有分组`、`次日重置待办状态`、`隐藏已完成的待办`、`排序`、`立即同步`、`从云端恢复列表`。

### 分组和待办

- 分组详情页顶部左侧关闭，标题居中偏左，右上是编辑图标；空状态文案为 `还没有待办哦`。
- 详情页底部固定输入区：左侧输入 `输入待办`，右下圆形 `+` 是添加当前输入内容。这个 `+` 只用于添加待办，和首页底部全局 `+` 不同。
- 任务保存后变成横向任务条：左侧圆形勾选，中心任务文本，右侧上下箭头/排序提示。
- 点击任务圆圈可完成/取消完成；完成后任务文字置灰，卡片内也置灰显示。
- 点击已完成任务会弹出底部动作面板，展示任务名、完成时间，以及操作：`自由计时`、`番茄计时`、`视频打卡`、`取消完成`、`设定目标`、`计划日期`、`删除`、`统计`。
- 分组卡三点菜单是面板式菜单，顶部三宫格为 `添加 / 重命名 / 颜色`，下面列表为 `统计`、`复制内容`、`每日重置`、`喵喵机打印`、`克隆`、`排序`、`分享`、`移至回收站`。
- 分组颜色选择是底部抽屉，标题 `选择颜色`，色块为圆形大按钮，当前色有发光边框和勾选。

### 全局 `+` 面板

- 底部中心 `+` 从任意页打开全屏动作面板，保留当前页作背景并叠加沉浸式背景图。
- 面板顶部有下拉箭头、设置、扫码；中部是今日概览卡：`今日计时`、`起床时间`、`今日数据`。
- 主动作卡：`自由计时`、`番茄计时`、`视频打卡`、`起床睡觉`；休息卡提供 `5分钟 / 10分钟 / 20分钟`。
- 面板可以上下拖动，像可拖拽的全屏/半屏层；下滑回到原位，不直接关闭。
- `自由计时` 不直接开始，而是弹出底部确认抽屉：标题 `开始自由计时`、名称输入框、右侧勾选按钮用于确认/确定计时名称。
- `番茄计时` 弹出底部配置抽屉：名称输入框、默认 `00 小时 25 分钟` 的滚轮选择器、`开始计时` 按钮。
- `视频打卡` 先弹业务权限说明：`需要相机权限`，按钮为 `取消 / 去设置`；不是一上来直接拉系统权限。
- `起床睡觉` 弹出底部选项滚轮/菜单：`起床 / 睡觉 / 取消`。
- 休息快捷项会直接进入全屏倒计时运行页，停止后返回面板，并会进入时间轴记录。

### 计时运行页

- 运行页为沉浸式背景图，中心大数字，类型名称在数字下方，可编辑名称。
- 底部是大圆形停止按钮；上方有横向工具条：`讨论`、`翻页时钟`、`白噪音`、`屏幕常亮`、锁/严格相关入口。
- 右上有扩展/全屏样式按钮。
- `我的 > 计时器` 会在当前页叠出一个小型浮动计时器卡片，包含计时时间、关闭、暂停、重置按钮。

### 统计页

- 顶部是日期范围胶囊，如 `今天 2026/6/18`，右侧有周纵览入口和更多按钮。
- 顶部功能卡包含 `补录` 和 `自定义`。
- 统计图表采用可折叠卡片：`饼图`、`专注时段`、`周纵览` 等；卡片右上有收起、合并、隐藏指示器等控制。
- 底部浮动分段为 `日 / 周 / 月 / 年`，切换后顶部范围和图表内容联动变化。
- 日期下拉进入整页日历视图，底部有 `颜色-时长` 图例。
- `自定义` 进入日期范围选择器，顶部有 `关闭 / 保存`，支持输入模式切换。
- `补录` 进入独立页面，字段包括计时名称、开始时间、结束时间、总时长、保存。
- 周纵览可全屏查看一周时间网格，按天和小时展示记录。

### 发现页

- 发现页当前不是本项目近期目标，但其信息架构可作为复杂列表参考。
- 顶部内部 Tab：`活动 / 自习室 / 消息 / 排行榜`。
- 活动页：公告卡在顶部，下面是实时动态流，条目包含用户、任务、状态、时间。
- 自习室页：还有二级横向分类，控件树显示 36 个分类；房间卡展示标题、简介、今日时长、人数、标签、是否需要密码。
- 消息页：入口列表包括 `统计数据`、`系统通知`、`点赞`、`新增关注`、`评论`。
- 排行榜页：顶部三列榜单卡，下面是学习室排名列表，标注每天更新。

### 我的页和设置

- 登录后顶部是账号卡：头像、昵称、User ID、二维码、关注/粉丝、等级进度、个人主页、设置。
- 内容区内部 Tab：`我的 / 时间轴`。
- `我的` 列表入口：会员中心、使用帮助、计时器、视频草稿箱、计时统计、进入翻页时钟、日历订阅服务、导出数据、倒数日、扫一扫、我的二维码、皮肤、背景图、分享给好友、设置。
- 上滑到底后最后一个入口是 `设置`，没有更多隐藏项。
- 时间轴按日期展示记录，条目包含开始/结束时间、类型、持续时长。例如休息记录显示 `休息 46秒 21:35 - 21:36`。
- 导出数据是内部页面，说明 `导出的数据会以xlsx格式保存在手机本地，可以使用Excel打开。`，按钮为 `导出待办数据` 和 `导出计时数据`。
- 我的二维码页展示个人二维码，底部有 `扫一扫` 和 `保存`。
- 皮肤页是横向预览轮播，直接预览计时运行界面和底部工具条。
- 背景图页有 `本地 / 画廊` Tab，图片网格包含空白、书本、晴天、森林、天竺葵、选择本地图片等。
- 设置页分组清晰：`个人信息`、`计时设置`、`个性化`、`其他`。
- 设置里的 `颜色` 是行内展开，不跳新页；展示 `自动` 和多枚圆形色彩按钮。建议本项目主题色入口只放一处，避免计时页和我的页重复。
- `计时设置` 页面包含：新一天开始时间、默认勾选、自动开启计时悬浮窗、白噪音音频设置、震动提醒、声音提醒、通知提醒、提前提醒时间。
- 设置下滑后还有：背景图、自定义格言、低配置模式、跟随系统字体缩放、实验性功能、关于、退出登录。

## 对本项目的近期借鉴点

1. 清单页优先改为 卡片式信息架构：顶部工具入口、空状态 CTA、分组卡三点菜单、底部详情输入栏。
2. 首页底部中心 `+` 做成全局动作面板，先提供自由计时、番茄计时、休息快捷项；视频打卡和起床睡觉可先作为占位或本地版入口。
3. 计时类型选择可以保留在计时页左上，但更推荐逐步转向全局 `+` 面板承载“开始什么计时”。
4. 主题色入口集中到 `我的 > 设置 > 个性化 > 颜色`，计时页不再放第二个主题入口。
5. 权限请求先用业务解释弹窗，再进入系统授权；避免启动时无上下文地请求存储权限。
6. 导出数据做成独立内部页，明确导出内容和格式，再触发保存/分享流程。
7. 统计页后续可以按 `日期范围 + 补录/自定义 + 可折叠图表 + 日周月年分段` 搭骨架。

## 2026-06-19 参考应用统计与计时补充整理

### 统计页图表交互

- 饼图卡片并不是纯展示：饼图区域可点击，`同名合并` 旁边的圆形切换控件可以切换饼图呈现形式。
- 切换后饼图会从“左侧饼图 + 右侧列表/标签”的布局，变成更大的居中圆环/饼图形式，标签嵌入图形中心或周边。
- 统计页下滑可见 `时间轴` 区块，条目把专注和休息记录都混排在同一日期下，包含开始时间、结束时间、名称、类型和时长。
- 时间轴里的 `补` 是可点击入口，会进入 `补录数据` 页面；页面包含计时名称、任务/分组芯片、开始时间、结束时间、总时长和保存动作。
- 底部 `更多图表 / 添加图表` 会进入图表市场式页面：两列卡片网格，图表项包括 `时间线图`、`小番茄`、`多彩饼图`、`矩形树图`、`漏斗图`、`时长比较`、`甘特图`、`玫瑰图`、`词云` 等，很多带会员标识。
- `图表管理 / 管理图表` 页面是列表管理结构：行项目包括 `饼图`、`周纵览`、`单周每日时长`、`专注时长`、`专注时段`、`作息时间`、`时间轴` 等；每行有图标、名称、`内置图表` 标记、可见性眼睛开关和拖拽排序柄。
- 借鉴点：本项目的统计页可以先做“内置图表列表 + 可见性开关 + 简单排序”的骨架，图表市场和 VIP 图表先不做；饼图样式切换可作为后续增强。

### 番茄计时与休息计时

- 番茄计时从全局 `+` 面板进入后会先弹出底部配置抽屉；空名称直接启动会提示 `请输入计时名称`。
- 番茄配置抽屉包含名称输入框、小时/分钟滚轮，默认时长为 `00:25`，底部主按钮为 `开始计时`。
- 番茄运行页的中心主数字是剩余倒计时，例如 `24:58`，不是自由计时那种向上累计时长。
- 番茄暂停后再停止会弹出确认面板，选项包括 `放弃本次计时(不记录时长)`、`提前完成并记录时长`、`取消`；这和自由计时停止即保存不同。
- 休息快捷项运行页使用杯子图标和剩余倒计时，停止后返回全局 `+` 面板，并出现 `计时完成` 一类反馈；观察过程中产生过一条短休息记录。
- 借鉴点：本项目应区分自由计时、番茄计时、休息计时的停止策略：自由计时可直接保存，番茄计时需要放弃/提前完成确认，休息计时可以弱记录或单独归类。

### 运行页工具与记录编辑

- 计时运行页工具条已确认可见入口包括：`讨论`、`翻页时钟`、`白噪音`、`屏幕常亮`，右侧还有锁定/严格模式相关入口但未稳定触发。
- 运行记录可进入 `编辑计时` 页面，结构包括名称字段、任务/分组芯片、`备注 / 心得`、开始时间、暂停总时长、结束时间，以及 `统计 / 删除 / 保存` 操作。
- 借鉴点：本项目近期可以只实现 `备注 / 心得` 和记录编辑保存；讨论、白噪音、严格模式先作为后续插件化功能，不进入当前 APK 主线。

## 2026-06-20 参考应用已办主页面补看

- 参考 App 当前前台包名为 `reference.app`，在竖屏模拟器中点击清单主页面左上角 `待办 ↔` 区域后，标题会直接切换成 `已办 ↔`；这不是下拉菜单，也不是分组详情页入口。
- `已办` 主页面结构不是单纯完成任务列表，而是三段：
  - 顶部 `统计概览` 卡，展示今日、本周、本月、总计。
  - 中部按分组/类型聚合的卡片，例如 `未分类 / 学习 / 工作 / 读书`，每张卡显示该类型已办数量。
  - 下部按日期组织的完成记录时间线，记录行展示待办名称、完成时间胶囊和分类入口。
- 点击已办页的分组卡会进入二级已办页面，例如 `未分类`，只看该类型/分组下的完成记录，顶部有返回和添加入口。

- 借鉴点：本项目的主页面 `待办 / 已办` 应保留直接切换；`已办` 页应先做统计概览、分组聚合、日期时间线和记录动作面板。云同步、会员、分类市场等不进入当前 APK 主线。
