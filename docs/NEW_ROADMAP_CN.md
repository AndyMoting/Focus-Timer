# Focus Timer 新路线图

日期：2026-06-19

定位：Android 优先、本地优先、参考 参考交互的专注计时 + 待办 + 统计工具。

这份文档用来把前面很多轮版本做过的东西重新梳成一条线。旧文档里有 W1、W2、W3、W4、V5、Codex 本地 APK 改造等阶段，容易看散；这里按“已经完成 / 当前状态 / 下一步路线”重新整理。

## 一句话方向

先把 APK 做成一个能日常用的本地版 Focus Timer：

- 不做登录。
- 不做云同步。
- 不做会员墙。
- 不做发现/社区。
- 学习 参考应用的信息架构和交互手感，但用自己的 Flutter + Riverpod + Drift 实现。

当前主结构：

1. 待办
2. 计时
3. 统计
4. 我的

## 新路线：从现在开始怎么走

### R0：整理和保护现场

目标：让当前成果不丢、不混乱、可交接。

要做：

- 保留当前脏工作区，不重置。
- 确认重要新增文件：
  - `docs/AGENT_HANDOFF.md`
  - `docs/Codexplan.md`
  - `lib/domain/models/task_plan_settings.dart`
  - `lib/presentation/screens/task_plan_screen.dart`
- 开只读 agent 时，只让它测试/审查，不让它改代码。
- 准备后续按功能拆提交。

完成标准：

- Claude 或新 agent 能看 `AGENT_HANDOFF.md` 接上。
- 普通 `git status` 不再显示临时截图/XML。
- 最新 APK 路径明确。

### R1：统计页手测和修补

目标：确认刚重构的统计页能真实使用。

要测：

- 日 / 周 / 月 / 年切换。
- 上一段 / 下一段导航。
- 补录记录。
- 补录后摘要、热力图、饼图、时间轴是否刷新。
- 图表管理隐藏/显示。
- 年视图热力图是否太密。
- 饼图图例和时间轴是否溢出。

要补：

- 时间轴记录编辑。
- 时间轴记录删除。
- 图表显隐持久化。
- 图表顺序管理。

完成标准：

- 用户能通过统计页看到自己刚补录或刚计时的数据。
- 统计页不再“一片白”或只像占位页。

### R2：计划模式收口

目标：把待办计划模式变成稳定可用的每日排程工具。

要测：

- 多分组任务进入全局计划。
- 拖到 00:00、08:30、23:30 等不同槽位。
- 移动计划条。
- 拖回左侧删除。
- 一键清空。
- 15/30/60/120 分钟槽位。
- 输入法弹起时待办输入框不再被顶得奇怪。

要补：

- 窄屏布局。
- 平板布局。
- 计划条颜色和长度微调。
- 计划记录编辑。

完成标准：

- 能像 reference app 那样把今天要做的待办拖到时间轴里。
- 用户不需要通过顺序猜时间。

### R3：计时闭环打磨

目标：计时不仅能跑，还要跑得顺手、停得明白、保存得安心。

要做：

- 自由计时停止后显示最近记录反馈。
- 番茄计时保存/放弃逻辑再确认。
- 快速休息是否进入统计，要给清晰规则。
- 运行页工具条整理。
- 起床睡觉和视频打卡先做本地可用，不碰云和社交。

完成标准：

- 开始计时、暂停、停止、保存、统计可完整闭环。
- 用户不会疑惑“这次到底有没有被记录”。

### R4：待办主页和分组体验整理

目标：让待办不只是能用，而是适合高频使用。

要做：

- 创建分组改底部抽屉。
- 分组卡片信息压缩。
- 任务动作面板。
- 分组菜单真实能力和占位能力分开。
- 考虑单条待办颜色。

完成标准：

- 新建分组、新建任务、完成任务、进计划模式都顺手。
- 菜单项没有“点了没反应”的感觉。

### R5：我的页和数据管理

### R7：液态玻璃视觉路线

目标：在核心 APK 稳定后，加入接近 iOS 26 Liquid Glass 的轻量视觉系统，但不牺牲流畅度。

设计原则：

- 高斯模糊只是底层材料之一，不把它理解成“全屏磨砂”。
- 玻璃感由高斯模糊、半透明底色、边缘高光、细阴影、背景对比控制和轻微动态响应共同组成。
- 优先用于底部导航、底部抽屉、悬浮工具条、小型菜单和统计图表控制层。
- 不在大面积列表、热力图主体、时间轴主体、输入页背景上铺满模糊。
- 正式版不卡才算通过；debug 包的卡顿只作为参考，不作为最终判断。

技术约束：

- Flutter 里优先封装统一 `GlassSurface`，集中控制 blur sigma、透明度、边框和降级策略。
- `BackdropFilter` 必须限制尺寸和裁剪范围，避免全屏或列表内大量重复 blur。
- 低端设备或掉帧场景可以退化为半透明色块 + 边框高光。
- 键盘弹起、计划模式拖拽、统计页滚动、Tab 切换是重点性能测试场景。

完成标准：

- release APK 在真机和模拟器上滚动、拖拽、键盘弹起都不明显掉帧。
- 视觉风格比当前更轻、更通透，但主信息仍清晰可读。
- 玻璃组件可复用，不在各页面散写临时样式。
